## Change 1 — Streak Card (replaces "Total Forks" stat card)

Replace the "Total Forks" stat card with a **Streak Card** that tracks the
user's daily task completion or GitHub push activity.

**Logic:**
- A "streak day" counts if the user completed at least 1 task OR made at
  least 1 GitHub push that calendar day
- Store streak data in localStorage under `devdash_streak`:
  `{ currentStreak, longestStreak, lastActiveDate, history: [] }`
- On app load, check if today's date matches `lastActiveDate`. If not,
  check if yesterday was active — if yes, streak continues. If not, reset
  to 0 and store the broken streak date.

**Visual:**
- Icon: a small flame or lightning bolt (use an SVG inline, no icon library)
- Stat number: current streak count in days
- Label: "DAY STREAK"
- Below the number, one line of humane copy that changes based on streak:
  - 0 days: "Start something today."
  - 1 day: "One step. Keep going."
  - 2–4 days: "Building momentum →"
  - 5–9 days: "You're in a rhythm now."
  - 10–19 days: "Two weeks of showing up. Rare."
  - 20–29 days: "This is discipline, not motivation."
  - 30+ days: "You've earned this. Don't stop."
- Subtle glow on the icon when streak > 7 (box-shadow pulse, CSS only)
- No confetti, no popups — keep it calm and understated

## Change 2 — Activity Page (new route: /activity)

Add a new page and sidebar link called **Activity**. This is a deep dive into
the authenticated user's own GitHub data — not a search tool.

The username is pulled from the existing GitHub connection (same username
already used on the Dashboard). Fetch from:
- `https://api.github.com/users/{username}/events?per_page=100`
- `https://api.github.com/users/{username}/repos?per_page=100`

**Sections on this page:**

### Commit Pulse (top section)
A contribution-style heatmap grid showing push activity.
- X-axis: last 52 weeks (columns), Y-axis: days of week (rows)
- Each cell is a small square, color intensity based on commit count that day:
  - 0 commits: near-transparent (base surface color)
  - 1–2: accent color at 30% opacity
  - 3–5: accent color at 60% opacity
  - 6+: accent color at full opacity
- Hovering a cell shows a tooltip: "3 commits · Mon 14 Apr"
- Build the grid purely from the events API data (PushEvent type)
- Include a time range selector: Last 30 days / Last 3 months / Last 6 months
  / Last year — this filters which portion of the grid is highlighted/active

### Code Stats (middle section)
Three cards in a row:
1. **Most active day** — which day of the week has the most pushes
   (calculated from events history). E.g. "Thursday"
2. **Avg commits/week** — total commits in last 30 days ÷ 4
3. **Top language** — language that appears most across all repos

### Repo Deep Dive (bottom section)
A sortable table of all repos with columns:
- Repo name (linked)
- Language (with a colored dot matching language color)
- Stars · Forks
- Last pushed (relative time: "3 days ago")
- Commit frequency badge: Active / Moderate / Dormant
  - Active: pushed in last 14 days
  - Moderate: pushed in last 60 days
  - Dormant: no push in 60+ days

Sortable by: Stars / Last pushed / Name (alphabetical)

## Implementation rules

1. **useFetch hook** — all API calls go through the existing useFetch hook.
   No raw fetch() calls in components.

2. **useMemo** — the repo filter on the GitHub page must use useMemo:
   ```js
   const filtered = useMemo(() =>
     repos
       .filter(r => selectedLangs.length === 0 ||
                    selectedLangs.includes(r.language))
       .filter(r => r.name.toLowerCase().includes(searchText.toLowerCase())),
     [repos, selectedLangs, searchText]
   )
   ```

3. **No design system changes** — do not change: font family, base font
   size, sidebar width, sidebar background color, top nav height, card
   border-radius, or the accent color. Every new element must feel like it
   was always part of the design.

4. ** use new dependencies** — you can install any new npm packages only when necessary.

5. **Loading and error states** — every new component that fetches must
   handle loading (spinner) and error (descriptive message). No blank
   screens.

6. **Streak card localStorage** — the streak must persist across sessions.
   Every time the app loads, recalculate streak from stored history before
   rendering.

7. **CSS transitions** — all new interactive elements (hover states, tab
   switches, progress bar fill, task completion flash) must use CSS
   transitions, not JavaScript-driven animations.

8. **Mobile baseline** — all new layouts must not break below 768px.
   The repo grid on GitHub page collapses to 1 column on mobile.
