## Context — how the username works

This is a personal dashboard. The username (`bolujxl`) is stored in
localStorage under the key `devdash_username` and loaded on mount.
The dashboard always opens directly to that user's data — no login,
no setup screen.

However there is a secondary use case: the user wants to temporarily
view a different GitHub profile. The existing `bolujxl` pill tag beside
the GitHub button is the trigger for this. We are keeping this pill
but making it functional, intentional, and obviously interactive.

The username stored in localStorage is the source of truth for all
GitHub fetches on the Dashboard. When it changes, the dashboard
re-fetches everything with the new username.

## Change 3 — Transform the username pill into an interactive profile switcher

**Do not remove the `bolujxl` pill.** Transform it into a functional
"switch profile" trigger. This is the mechanism that lets the user load
a different GitHub username into the dashboard.

### Pill default state (not editing):

```jsx
<button className="hero-username-pill" onClick={() => setEditing(true)}>
  <span className="pill-edit-icon">✎</span>
  {username}
</button>
```

**CSS:**
```css
.hero-username-pill {
  display: inline-flex;
  align-items: center;
  gap: 5px;
  font-size: 0.75rem;
  font-weight: 500;
  padding: 4px 12px;
  border-radius: 20px;
  border: 1px solid var(--card-border-color);
  color: var(--color-text-secondary);
  background: transparent;
  cursor: pointer;
  transition: border-color 0.15s ease, color 0.15s ease;

.hero-username-pill:hover {
  border-color: var(--accent-color);

  font-size: 0.7rem;

  transition: opacity 0.15s ease;

  opacity: 1;

```

The `✎` pencil icon signals editability without a tooltip or label.
On hover, border shifts to accent color — the pill comes alive.

### Pill editing state:

When clicked, `setEditing(true)` — the pill is replaced inline by a
compact input in the exact same position:

```jsx
{editing ? (
  <div className="hero-username-input-wrap">
    <input
      className="hero-username-input"
      type="text"
      value={inputValue}
      onChange={e => setInputValue(e.target.value)}
      onKeyDown={handleUsernameKeyDown}
      onBlur={handleUsernameBlur}
      autoFocus
      placeholder="github username"
      spellCheck={false}
    />
  </div>
) : (
  <button className="hero-username-pill" onClick={() => setEditing(true)}>
    <span className="pill-edit-icon">✎</span>
    {username}
  </button>
)}
```

**Input CSS:**
```css
.hero-username-input-wrap {
  display: inline-flex;
  align-items: center;
}
.hero-username-input {
  font-size: 0.75rem;
  font-weight: 500;
  padding: 4px 12px;
  border-radius: 20px;
  border: 1px solid var(--accent-color);
  background: transparent;
  color: var(--color-text-primary);
  outline: none;
  width: 140px;
}
.hero-username-input::placeholder {
  color: var(--color-text-secondary);
  opacity: 0.5;
}
```

The input adopts the pill shape of the button it replaces — the
transition feels like the pill became editable, not like a new element
appeared. No layout shift.

### Interaction logic:

Add these state variables to the Dashboard or hero card component:

```js
const [editing, setEditing]       = useState(false)
const [inputValue, setInputValue] = useState('')
```

**On Enter key — confirm and switch:**
```js
function handleUsernameKeyDown(e) {
  if (e.key === 'Enter') {
    const trimmed = inputValue.trim()
    if (trimmed && trimmed !== username) {
      localStorage.setItem('devdash_username', trimmed)
      window.location.reload()
      // Reload re-runs all dashboard fetches with the new username.
      // TODO: replace with username context when global state is added.
    }
    setEditing(false)
    setInputValue('')
  }
  if (e.key === 'Escape') {
    // Discard — do not save
    setEditing(false)
    setInputValue('')
  }
}
```

**On blur — discard silently:**
```js
function handleUsernameBlur() {
  // User clicked away without confirming — abandon the edit
  setEditing(false)
  setInputValue('')
}
```

**Why `window.location.reload()`:**
The dashboard reads username from localStorage on mount. A full reload
is the simplest way to re-run all child fetches with the new value
without prop drilling or a context update. This is intentional for now.

### Terminal bar path becomes dynamic:

The bar currently shows `~/bolujxl` hardcoded. Make it reflect the
active username:

```jsx
<span className="terminal-path">~/{username}</span>
```

Where username is initialized from localStorage:
```js
const [username] = useState(
  () => localStorage.getItem('devdash_username') || 'bolujxl'
)
```

When the page reloads after a username switch, the terminal bar
automatically shows the new name.
